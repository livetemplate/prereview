Payments guide

The charge flow moves money in three hops: Charge → Gateway → Refund. Each hop is idempotent on the orderID, so a retried request never double-charges.

Retry policy

Transient gateway failures should retry with exponential backoff, capped at five attempts. Always thread the caller's context so a cancelled request stops retrying:

for attempt := 0; attempt < 5; attempt++ {
    if err := submit(ctx, charge); err == nil { return nil }
    backoff(attempt)
}
Note: refunds bypass the retry loop — a failed refund is surfaced to the operator immediately.
